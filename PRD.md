# PRD — ASI Chatroom (Chatroom-ASI) v2.0
> 产品需求文档 v2.0 | 2026-05-08
> 核心关键词：模拟、辩论、变聪明、Skill、MCP、直接操作

---

## 1. 产品定位（一句话）

**多智能体辩论推演平台** — Agent 在房间中模拟真实辩论，从经验中学习变聪明，通过 Skill + MCP 直接执行操作。

---

## 2. 核心概念（6个关键词）

### 2.1 模拟（Simulation）
- 多Agent在房间中模拟真实场景辩论
- 每轮所有Agent发言，有明确角色分工
- Phase系统：信息收集 → 观点表达 → 冲突辩论 → 共识合成 → 决策输出

### 2.2 辩论（Debate）
- Agent 不是各自独立发言，而是**针对同一议题交锋**
- 支持：支持/反对/中立 三种立场
- 风险官可一票否决（confidence > 80 时）
- 共识度过低（<30%）自动触发回溯检查点

### 2.3 变聪明（Evolution / Learning）
**这是灵魂功能。**
- Agent 有**跨房间长期记忆**（Long-term Memory）
- 每次推演后自动提取经验教训：
  - "上次支持微服务架构被风险官否决，原因是运维成本高"
  - "上次数据分析师的计算被证实准确"
- 下次推演同一话题时，Agent 自动调整策略
- 学习粒度：单Agent独立学习，不共享（避免集体偏见）

### 2.4 Skill（可插拔技能）
- Agent 能力抽象为 Skill 包，按需加载
- 内置 Skill：
  - `debate` — 辩论基础能力（默认自带）
  - `calculation` — 数据计算、概率评估
  - `risk_assessment` — 风险评估、合规检查
  - `cost_analysis` — 成本分析、ROI计算
- Skill 可组合：Agent 角色决定加载哪些 Skill
- 未来支持用户自定义 Skill

### 2.5 MCP（工具连接）
- Agent 通过 MCP Server 连接外部工具
- 推演中 Agent 可调用工具获取信息：
  - "查一下当前云服务价格" → 调用价格查询 MCP
  - "看看GitHub上这个项目的star数" → 调用 GitHub MCP
- **人类在回路**：Agent 提议调用工具 → 用户确认 → 才执行

### 2.6 直接操作（Action / Execution）
- 推演达成共识后，**一键执行**
- 操作类型（预设 Action）：
  - 发飞书消息 / 创建飞书任务
  - 发送邮件
  - 生成报告（Markdown/PDF）
  - 调用内部API（如创建工单）
- **必须人类确认**：Agent 生成 Action 草稿 → 用户审核 → 点"执行"

## 4. 数据流

```
用户创建房间 → 选择话题 + Agent角色 + Skill包
                    ↓
              Agent 初始化（加载Skill、设置System Prompt）
                    ↓
              推演开始 → 每轮：所有Agent发言 + 辩论交锋
                    ↓
              阶段结束 → 计算共识度 + 保存消息
                    ↓
              一轮结束 → Agent 提取经验 → 更新长期记忆
                    ↓
              推演完成 → 生成结论 + Action草稿（可选）
                    ↓
              用户确认 → 执行Action（发飞书/邮件/生成报告）
```

---

## 5. 界面设计原则

**极简主义**：
- 默认只显示：聊天流 + 底部控制条 + 顶部轮次指示
- 高级功能（检查点、Skill配置、MCP工具）→ 折叠到侧边栏/设置里
- 推演完成 → 弹出结果面板，引导"导出/Fork/执行Action"

---

## 6. 非功能需求

| 项 | 要求 |
|----|------|
| 响应速度 | Agent发言延迟 < 5秒（DeepSeek API） |
| 并发 | 支持5个房间同时推演 |
| 数据持久 | 所有消息、Agent状态、学习记录持久化 |
| 安全 | Action执行必须人类确认，不可自动执行 |
